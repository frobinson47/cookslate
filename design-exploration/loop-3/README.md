# Loop 3: Command Palette & Empty States

## Direction
Two dimensions that Loop 2 explicitly deferred. Command palette builds on the filtering/search patterns from Loop 2, and empty states address the biggest gap in the current UI's emotional design — the moments when there's nothing to show.

## Dimensions explored
1. **Command palette** — global Cmd+K overlay, result ranking, action shortcuts
2. **Empty states** — zero-data, zero-results, first-run, and error states across key screens

## What this loop builds on from Loops 1-2
- Loop 1 tokens: uses the color system, motion curves, shadow system
- Loop 1 navigation IA: command palette integrates with the collapsible rail (search icon triggers it)
- Loop 2 filtering/search: command palette surfaces the same faceted filters as quick actions, not a separate search system

## What this loop rejected
- **Density toggle** was deferred from Loop 2 — still deferring to Loop 4 where it pairs better with a home page grid mockup
- **Data visualization** — not yet, needs the recipe detail and home page settled first
- Did not build a full mockup of the command palette — an interaction sketch + spec is more useful at this stage since it's a behavior-heavy component

## Deliverables
1. `command-palette.md` — interaction design spec for Cmd+K command palette
2. `empty-states.html` — HTML/CSS mockup showing 5 empty state variations
