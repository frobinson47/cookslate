# Loop 2: Recipe Detail Hero Mockup & Filtering/Search

## Direction
Apply Loop 1's design tokens to the most important screen in the app — the recipe detail page. Simultaneously explore how filtering and search should work on the home page, since that's the gateway to every recipe.

## Dimensions explored
1. **Filtering & Search** — current tag-only filtering vs. faceted search with saved views
2. **Recipe Detail Layout** — applying tokens, improving visual hierarchy, tightening spacing

## What this loop builds on from Loop 1
- Uses the proposed color tokens (cream scale, terracotta/sage semantic roles)
- Uses the typography decisions (Playfair Display for recipe titles only, Nunito for everything else)
- Uses warm-tinted shadows from the token system
- Incorporates the navigation IA recommendation (collapsible rail sidebar, giving more horizontal space to content)

## What this loop rejected
- Did not explore command palette yet — search and filtering needed first to establish the information architecture patterns that command palette would accelerate
- Did not explore density toggle — recipe detail is inherently a comfortable-density page; density toggle is more relevant for grid/list views (Loop 3)

## Deliverables
1. `recipe-detail-mockup.html` — single-file HTML/CSS mockup of the proposed recipe detail page
2. `filtering-search.md` — current vs proposed filtering and search patterns
