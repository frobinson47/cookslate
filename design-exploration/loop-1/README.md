# Loop 1: Foundation — Design Tokens & Navigation/IA

## Direction
Establish the design token foundation and evaluate navigation architecture. Before exploring surface-level aesthetics, we need to understand the structural bones of the app and define the raw materials (color, type, spacing, motion) that every future loop builds on.

## Dimensions explored
1. **Design Tokens** — audit current palette, propose a refined system with semantic layers
2. **Navigation / IA** — evaluate current sidebar+header+bottom-nav pattern against reference products

## What this loop rejected
- Skipped visual polish, component-level exploration, and empty states — those need tokens and IA settled first.
- Did not explore dark-mode-first (the prompt suggests it, but Cookslate's domain is food/cooking — warm, light, inviting is the correct default; dark mode is the variant, not the base).

## Deliverables
1. `design-tokens.json` — proposed token system
2. `navigation-ia.md` — current vs proposed navigation architecture
